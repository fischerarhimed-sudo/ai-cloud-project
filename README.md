# AI Cloud Project — Cloud-Native AI Application

## Описание проекта

Данный проект представляет собой **cloud-native веб‑приложение**, реализованное с использованием **FastAPI**, **Docker** и облачного PaaS‑сервиса **Render**. Приложение предоставляет REST API для генерации текста с использованием внешнего AI‑сервиса (Hugging Face) и содержит механизм отказоустойчивости (fallback), что соответствует практикам промышленной разработки.

Проект выполнен в учебных целях и демонстрирует полный цикл разработки, контейнеризации и деплоя облачного приложения без необходимости привязки банковской карты.

---

## Цель работы

Целью проекта является:

* изучение принципов **cloud-native архитектуры**;
* освоение контейнеризации с помощью **Docker**;
* развертывание API‑сервиса в облаке;
* интеграция внешнего AI‑сервиса через HTTP API;
* обеспечение стабильной работы приложения при недоступности внешних сервисов.

---

## Используемые технологии

* **Python 3.10**
* **FastAPI** — REST API фреймворк
* **Docker** — контейнеризация приложения
* **Render (Free Web Service)** — облачный хостинг
* **Hugging Face API** — AI inference (внешний сервис)
* **Git / GitHub** — система контроля версий

---

## Архитектура приложения

Приложение построено по принципам cloud-native:

* stateless API
* конфигурация через переменные окружения
* контейнерная упаковка
* внешний AI как managed‑service

### Логика работы

1. Клиент отправляет POST‑запрос на `/generate`.
2. Приложение пытается получить ответ от Hugging Face API.
3. Если внешний AI недоступен (ошибка 4xx/5xx), используется **fallback‑ответ**.
4. Клиент всегда получает корректный HTTP‑ответ.

## API Endpoints

### `GET /`

Проверка работоспособности сервиса.

**Ответ:**

```json
{
  "status": "AI Text Generator is running"
}
```

---

### `POST /generate`

Генерация текста на основе пользовательского запроса.

**Request body:**

```json
{
  "text": "Explain cloud computing in simple words"
}
```

**Response (успешный):**

```json
{
  "generated_text": "Cloud computing means using computers over the internet...",
  "source": "huggingface"
}
```

**Response (fallback):**

```json
{
  "generated_text": "Cloud computing means using computers and servers over the internet...",
  "source": "fallback"
}
```

## Запуск проекта локально

### 1. Клонирование репозитория

```bash
git clone https://github.com/fischerarhimed-sudo/ai-cloud-project.git
cd ai-cloud-project
```

### 2. Сборка Docker‑образа

```bash
docker build -t ai-cloud-project .
```

### 3. Запуск контейнера

```bash
docker run -p 8000:8000 ai-cloud-project
```

Swagger UI будет доступен по адресу:

```
http://localhost:8000/docs
```

## Деплой в облаке

Приложение задеплоено в **Render (Free Web Service)**:

* автоматический билд из GitHub
* контейнерный запуск
* HTTPS по умолчанию
* не требуется привязка банковской карты

**URL сервиса:**

```
https://ai-cloud-project.onrender.com
```

---

## Обработка ошибок и отказоустойчивость

В проекте реализован механизм fallback:

* если внешний AI‑сервис недоступен;
* если происходит ошибка сети или API;

приложение возвращает заранее подготовленный ответ, что обеспечивает стабильность сервиса.
